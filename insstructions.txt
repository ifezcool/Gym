In pages/combined.py, the search_enrollee callback currently reads q2_data and q4_data from store-q2 and store-q4 Dash stores as State inputs. This causes a bug in multi-worker deployments where different gunicorn workers handle different requests — the worker that runs the search may never have populated its local cache, so the stores arrive empty or stale, causing member searches to return nothing or show outdated results.
The fix is to remove the dependency on those two stores entirely from this callback, and instead fetch the data fresh inside the function body using the already-existing cached_read_sql function with query_ps_q2 and query_ps_q4 — both of which are already defined in the file. The result of the query_ps_q4 fetch should be converted to a list of dicts to replace the q4_data variable that the rest of the function already uses. The filled_df variable should come directly from the fresh fetch rather than from pd.DataFrame(q2_data).
Do not modify any other part of the function body or any other callbacks in the file.
