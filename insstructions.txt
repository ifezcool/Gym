 In the update_pa_code callback, a pyodbc.DataError with code 22007 is raised when clicking Proceed in the Contact Center view. The error is: Conversion failed when converting date and/or time from character string. The error occurs at the cursor.execute(...) call inside update_pa_code, specifically on the WHERE MemberNo = ? AND date_submitted = ? clause.
Root cause: date_submitted is read from target_row['date_submitted'], where target_row comes from a DataFrame built from q2_data. q2_data is a dcc.Store that was populated by serialising a DataFrame to .to_dict('records'). JSON serialisation converts all pandas Timestamp and datetime objects to strings. So by the time update_pa_code runs, date_submitted is always a string, never a pd.Timestamp or dt.datetime. The existing isinstance check therefore never matches, the conversion is skipped, and the raw string — which may be in ISO format with a T separator, or may contain fractional seconds — is passed directly to pyodbc as the date_submitted parameter. SQL Server's ODBC driver cannot reliably parse arbitrary datetime strings and raises 22007.
Fix: Replace the existing isinstance conversion block for date_submitted with a robust parse that handles all possible input types — string, Timestamp, and datetime — and converts to a Python datetime object before passing to cursor.execute. Specifically: if date_submitted is already a dt.datetime, use it as-is; if it is a pd.Timestamp, call .to_pydatetime(); if it is a string, parse it with pd.to_datetime(date_submitted).to_pydatetime() which handles both "2025-03-15 09:23:41" and "2025-03-15T09:23:41.123456" formats. Wrap the parse in a try/except and return a dbc.Alert with a clear error message if parsing fails rather than letting the exception propagate.
Constraints:

Only change the date_submitted conversion logic inside update_pa_code — do not touch any other part of the callback
Do not change the SQL query itself
Do not change how pa_tests is handled or joined
The fix is three to five lines maximum — this is a targeted type-coercion patch, not a refactor
